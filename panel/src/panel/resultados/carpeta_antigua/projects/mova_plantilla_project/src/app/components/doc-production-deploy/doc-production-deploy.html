Información Información sobre despliegue en producción.

Antes de lanzar una aplicación MOVA en producción ya sea en modo webapp o móvil, hay que revisar varios aspectos:

Archivo config.xml:

  • id="org.madrid.MOVA.mova2App": poner el nombre de paquete que proporcione el departamento de Arquitectura
  • <name> MOVA </name>: poner el nombre de aplicación que corresponda
  • <description> Framework MOVA para el desarrollo de aplicaciones móviles. </description>: poner la descripción que corresponda
  • <variable name="URL_SCHEME" value="org.madrid.MOVA.mova2App" />: si no se ha eliminado el plugin cordova-plugin-customurlscheme (porque se usa el broker o porque es necesario para alguna funcionalidad propia de la app) habrá que poner el nombre de paquete que proporcione el departamento de Arquitectura

Archivo config-default.json:

  • "appId": poner el identificador que proporciona Arquitectura
  • "appVersion": poner la versión que corresponda, si es la primera vez que se lanza: 1.0.0
  • "moduloFuncional": poner el valor que corresponda
  • "moduloTecnicoIos": poner el valor que corresponda
  • "moduloTecnicoAndroid": poner el valor que corresponda
  • "moduloTecnicoWebapp": poner el valor que corresponda
  • "piwikId": poner el valor que proporciona Arquitectura. Si no tenemos dejarlo vacío

Textos de protección de datos:

Hay que cambiar los textos de protección de datos que vienen por defecto en la vista data-protection-view por los correspondientes a nuestra app.

Documentación del componente: doc-protection-view
