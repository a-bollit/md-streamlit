Información Información environment.ts, environment.val.ts y environment.prod.ts.

Estos archivos permiten definir la configuración por entorno (desarrollo/validación/producción) de la app.

Para configurar el aspecto visual general de la app (estilo rojo/azul, menú lateral, header...) ver la documentación de config-default-main-frame.json

Para configurar los aspectos generales de la app ver la documentación de config-default.json

Configuración Configuración environment.ts, environment.val.ts y environment.prod.ts.

En estos archivos se configuran las variables que varían según el entorno: desarrollo, validación y producción. Así, en el archivo environment.ts se configuran los valores para desarrollo, en el archivo environment.val.ts se configuran los valores para validación y en el archivo archivo environment.prod.ts se configuran los valores para producción.

  • mova:
    • environment - Variable para definir el entorno (DES/VAL/PRO). No se debe tocar.
    • brokerView (documentación broker-view):
      • autoRedirect - Variable para definir si queremos que el sistema broker-view redirija automáticamente al volver de la obtención del ticket.
    • initView (documentación init-view)
      • autoRedirect - Variable para definir si queremos que nos redirija automáticamente a la vista correspondiente al hacer el chequeo de nueva versión.
      • skipVersionRequirementsCheck - Variable para definir si no queremos que la app haga el chequeo de nueva versión.
      • showVersionNewsWeb - Variable para definir si queremos mostrar las novedades de la versión en modo webapp.
      • showVersionNewsMobile - Variable para definir si queremos mostrar las novedades de la versión en modo móvil.
    • loginView (documentación login-view)
      • dametokenSystemEnabled - Variable para definir si queremos mostrar el botón de usuario y contraseña - osb en la vista login-view.
      • dametokenButtonLabel - Variable para definir el texto del botón de usuario y contraseña - osb en la vista login-view.
      • dametokenInfoLabel - Variable para definir el texto explicativo al elegir el sistema usuario y contraseña - osb en login-view.
      • fingerprintAuth - Variable para definir si queremos acceso biométrico.
      • wso2SystemEnabled - Variable para definir si queremos mostrar el botón de WSO2 en la vista login-view.
      • wso2ButtonLabel - Variable para definir el texto del botón de WSO2 en la vista login-view.
      • wso2InfoLabel - Variable para definir el texto explicativo al elegir el sistema WSO2 en login-view.
      • autologinSystemEnabled - Variable para definir si queremos mostrar el botón de autologin - osb en la vista login-view.
      • autologinButtonLabel - Variable para definir el texto del botón de autologin - osb en la vista login-view.
      • autologinInfoLabel - Variable para definir el texto explicativo al elegir el sistema autologin - osb en login-view.
      • autologinWso2SystemEnabled - Variable para definir si queremos mostrar el botón de autologin - WSO2 en la vista login-view.
      • autologinWso2ButtonLabel - Variable para definir el texto del botón de autologin - WSO2 en la vista login-view.
      • autologinWso2InfoLabel - Variable para definir el texto explicativo al elegir el sistema autologin - WSO2 en login-view.
      • urlPasswordReset - Variablesssss para definir la url del link de recuperación de la contraseña en la pantalla de login.
      • urlBottomText - Variable para definir el texto del link de la parte baja de la pantalla de login.
      • urlBottomUrl - Variable para definir la url del link de la parte baja de la pantalla de login. Permite definir una url externa o el nombre de una ruta de la aplicación.
      • urlCornerIcon - Variable para definir el icono de material del link de de la esquina de la pantalla de login.
      • urlCornerUrl - Variable para definir la url del link de la esquina superior izquierda de la pantalla de login. Permite url externa o interna de la app. Sirve para crear una página de ayuda referente al login. Si se deja urlCornerUrl en blanco desaparecerá el icono
      • placeholder
        • user - Placeholder para el usuario en la pantalla de login
        • password - Placeholder para la contraseña en la pantalla de login
    • authenticationService documentación broker-view y documentación login-view
      • authenticationDefaultSystem - Variable para definir el sistema de autenticación que queremos: 'autologin', 'wso2' o 'dameToken'.
      • authenticationSaveLastLoginSystem - Permite definir qué sistema de autenticación aparecerá seleccionado en la pantalla de login en caso de poder escoger varios. Si el valor es true aparecerá seleccionado el último escogido. Si el valor es false aparecerá seleccionado el definido en la variable authenticationDefaultSystem.
      • authenticationSaveInSessionWeb - Permite escoger si, en modo web, queremos que las credenciales se almacenen en sessionStorage en vez de localStorage.
      • authenticationSaveInSessionMobile - Permite escoger si, en modo móvil, queremos que las credenciales se almacenen en sessionStorage en vez de localStorage.
      • dameTokenConfig:
        • uriBase - Variable para definir la url base del sistema dameToken. No se debe tocar.
        • uriEliminaToken - Variable para definir la url de eliminación del token al hacer logout. No se debe tocar.
        • authenticationSystem - Variable para definir el tipo de usuario del sistema dameToken. No se debe tocar.
        • expirationTime - Variable para definir el tiempo de expiración de los token del sistema dameToken.
        • httpHeaderTokenAuthAlways - Variable para definir si queremos pasar la cabecera token-auth en las llamadas http cuando estamos logados con el sistema dameToken o autologin.
        • technicalModules - Variable para definir un array con los módulos técnicos que podemos usar en la app.
        • authenticationTechnicalModuleIndex - Variable para definir qué módulo del array de módulos técnicos es el principal.
      • wso2Config:
        • uriBase - Variable para definir la url base del sistema WSO2. No se debe tocar.
        • uriRevoke - Variable para definir la url de revocación de tokens del sistema WSO2. No se debe tocar.
        • consumerKey - Variable para definir la clave pública del sistema WSO2 para nuestra app.
        • consumerSecret - Variable para definir la clave pública del sistema WSO2 para nuestra app.
        • customDomain - Variable para definir el customDomain de wso2.
        • customTenant - Variable para definir el customTenant de wso2.
        • scopes - Variable para definir el array de scopes relativos al sistema WSO2.
      • autologinConfig:
        • appRedirect - Variable para definir el identificador de la app para poder volver desde el broker en móvil.
        • uriBase - Variable para definir la url base del sistema autologin. No se debe tocar.
        • uriLogout - Variable para definir la url de logout de dameToken. No se debe tocar.
        • uriRedirectSite - Variable para definir la url del sitio web intermedio en el uso de autologin. No se debe tocar.
        • paramIosSS - Variable para definir el subservicio de la plataforma iOS, el parámetro ss, de la URL de Autologin.
        • paramIosPass - Variable para definir la contraseña para el servicio de la plataforma iOS, el parámetro pass, con la clave de la App para pasar por la URL de Autologin.
        • paramAndroidSS - Variable para definir el subservicio de la plataforma iOS, el parámetro ss, de la URL de Autologin.
        • paramAndroidPass - Variable para definir la contraseña para el servicio de la plataforma iOS, el parámetro pass, con la clave de la App para pasar por la URL de Autologin.
        • paramWebAppSS - Variable para definir el subservicio en WebApp, el parámetro ss, de la URL de Autologin.
        • paramWebAppPass - Variable para definir la contraseña para el servicio en WebApp, el parámetro pass, con la clave de la App para pasar por la URL de Autologin.
        • paramWebappOkUrl - Variable para definir otra url de vuelta del sistema Autologin. No se debe tocar.
        • paramWebappOkUrlLogout - Variable para definir la url de vuelta del sistema Autologin al hacer logout. No se debe tocar ya que se utilizará el estado actual en el que se encuentre la App a la hora de hacer logout
        • autologinOsb:
          • uriGetToken - Variable para definir la url de obtención de token de dameToken. No se debe tocar.
          • uriValidaTicket - Variable para definir la url para validar tokens de dameToken. No se debe tocar.
        • autologinWso2:
          • consumerKeyAutologin - Variable para definir la clave pública del sistema autologin - WSO2 para nuestra app.
          • consumerSecretAutologin - Variable para definir la clave pública del sistema autologin - WSO2 para nuestra app.
          • uriGetUserData - Variable para definir la url de obtención de datos del usuario con autologin - WSO2.
    • backendService
      • uriCheckVersion - Variable para definir la url base para obtener la versión de la app. No se debe tocar.
      • uriVersionNews - Variable para definir la url base para obtener las novedades de la versión. No se debe tocar.
      • uriScores - Variable para definir la url para valoraciones y comentarios de la app. No se debe tocar.
    • interceptorService
      • apiKeys - Variable para definir un array con las apiKeys necesarias para nuestra app.
    • fileUploaderComponent (documentación mv-file-uploader)
      • authorizationHeader - Variable para definir la cabecera de autenticación del rest.
      • uriUpload - Variable para definir la url de subida del archivo.
      • uriTokenBase - Variable para definir la url de obtención del token.
    • notificationService (documentación notificaciones push)
      • uriBase - Variable para definir la url base de la api de notificaciones de Madrid Digital. No se debe tocar.
      • alwaysShowIosTemplate - Variable para definir si queremos mostrar la vista de las notificaciones en iOS.
      • appKey - Variable para definir la clave de las notificaciones push.
      • browserServiceUri - Variable para definir la url del servicio de notificaciones push de Phonegap. No se debe tocar.
      • environment - Variable para definir el entorno de las notificaciones push.
      • receivedLimit - Variable para definir el número de notificaciones que se almacenarán como máximo en localStorage.
    • recaptchaService (documentación recaptcha)
      • uriUtilSite - Variable para definir la url para el uso de captcha en móvil. No se debe tocar.
      • siteKey - Variable para definir la clave para el servicio de recaptcha de Google.
    • localStorageService (documentación localStorageService)
      • CryptP - Variable para definir la password para encriptar los datos de localStorage.
      • mainNodeSuffix - Variable para definir un sufijo que se añadirá al nombre del nodo de localStorage. Si no se añade nada, el nombre del nodo de memoria local será el módulo funcional que hayamos definido.
    • sessionStorageService (documentación sessionStorageService)
      • CryptP - Variable para definir la password para encriptar los datos de sessionStorage.
      • mainNodeSuffix - Variable para definir un sufijo que se añadirá al nombre del nodo de sessionStorage. Si no se añade nada, el nombre del nodo de memoria local será el módulo funcional que hayamos definido.
