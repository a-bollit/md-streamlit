Información Información del archivo config-default.json.

Este archivo permite definir la configuración general de la app.

Para configurar el aspecto visual general de la app (estilo rojo/azul, menú lateral, header...) ver la documentación de config-default-main-frame.json

Para configurar aspectos según el entorno (desarrollo/validación/producción) ver la documentación de environment.ts, environment.val.ts y environment.prod.ts

Configuración Configuración del archivo config-default.json.
  • mova
    • appName - Nombre corto de la app. Aparece en el header (si no se sobrescribe con otro texto), en login-view, init-view...
    • appDescription - Descripción de la app. Aparece en el header (si no se sobrescribe con otro texto), en login-view, init-view...
    • appId - Identificador de la app.
    • appVersion - Versión de la app.
    • appVersionFramework - Versión de MOVA usada. No se debe tocar ya que se establece automáticamente mediante código interno.
    • appVersionCordovaPluginLib - Versión de la librería de plugins usada. No se debe tocar ya que se establece automáticamente mediante código interno.
    • compilationDate - Fecha de la última compilación de la app. No se debe tocar ya que se establece automáticamente mediante código interno.
    • dialogErrorHttp:
      • showDetail - Variable para mostrar el detalle del error en caso de fallo de llamadas http por defecto o no.
      • loginButton - Variable para mostrar el botón de login del error en caso de fallo de llamadas http por defecto o no.
    • isDebug - Variable para determinar si la app está en modo debug o release. No se debe tocar.
    • loadingHttpAlways - Variable para mostrar el loading por defecto o no.
    • landingViewRoute - Variable para definir la ruta de la vista de presentación de la app.
    • loginViewRoute - Variable para definir la ruta de la vista de login de la app. En principio no se debería tocar.
    • mainViewRoute - Variable para definir la ruta de la vista principal de la app.
    • moduloFuncional - Variable para definir el módulo funcional de la app.
    • moduloTecnicoIos - Variable para definir el módulo técnico para iOS.
    • moduloTecnicoAndroid - Variable para definir el módulo técnico para Android.
    • moduloTecnicoWebApp - Variable para definir el módulo técnico para WebApp.
    • moduloTecnicoWindows - Variable para definir el módulo técnico para Windows.
    • piwikId - Variable para definir el identificador de piwik (Matomo).
    • textAccessConditions - Variable para definir el texto del diálogo de las condiciones de acceso de la app.
    • titleAccessConditions - Variable para definir el título del diálogo de las condiciones de acceso de la app.
    • maxWidthAccessConditions - Variable para definir el ancho máximo del diálogo de de las condiciones de acceso de la app.
    • urlContact - Variable para definir la url de contacto de la Comunidad de Madrid.
    • urlLegal - Variable para definir la url de los aspectos legales de la Comunidad de Madrid.
    • urlPortals - Variable para definir la url de los portales de la Comunidad de Madrid.
    • urlVisorVgcm - Variable para definir la url del visor de mapas de VGCM.
  • app
    • // Sección para definir las variables del negocio de la app que no varían según el entorno.
