Dependencias del API GIS

El API GIS está preparada para para poder integrar diversas librerías GIS. Por ese motivo, es responsabilidad del integrador incluir dichas librerías y sus dependencias dentro de las aplicaciones que usen el API GIS. De esta forma, se evita incluir librerías muy pesadas que no vayan a emplearse.


IMPORTANTE

Actualmente el API GIS solo permite la integración de OpenLayers (v. 7.1.0).

Para poder emplear OpenLayers dentro del API GIS, el integrador deberá instalar (si así lo requiere su aplicación) las siguientes dependencias:

  • ol <versión 7.1.0 >. Para su instalación, deberá situarse en el directorio del proyecto y con la consola ejecutar "npm install ol@7.1.0".
  • ol-mapbox-style <versión 9.2.2 >. Para su instalación, deberá situarse en el directorio del proyecto y con la consola ejecutar "npm install ol-mapbox-style@9.2.2".
  • own-geostyler-openlayers-parser. Para su instalación, deberá situarse en el directorio del proyecto y con la consola ejecutar "npm install http://e_jpb71@gitlab.madrid.org/GIS_open/GIS_DIFUSION.git#own-geostyler-openlayers-parser".
