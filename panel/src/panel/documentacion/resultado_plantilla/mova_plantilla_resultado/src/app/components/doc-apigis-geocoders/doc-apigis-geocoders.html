Documentación de apigis.Geocoder

La geocodificación es el proceso de transformar una descripción de una ubicación en una ubicación de la superficie de la Tierra. El API GIS permite integrar la clase apigis.Geocoder para integrar el Geocoder de CartoCiudad o de OpenStreetsMap relizar procesos de geocodificación directa e inversa.

En esta página se muestra la forma en la que podemos construír un objeto de este tipo y los métodos disponibles para trabajar con él.

Constructor apigis.Geocoder

Se puede instanciar un geocoder mediante la clase apigis.Geocoder.

new ApiGeocoder(type)

  • type <object > Objeto con el tipo de provider ['Cartociudad','OSMNominatim'].

Se puede consultar información más detallada y ejemplos sobre el uso de apigis.Geocoder en el documento de Arquitectura sobre el API GIS.

Métodos Geocoder de Cortociudad Métodos disponibles en el Geocoder de Cortociudad (type: 'Cartociudad').

Se muestran a continuación los métodos disponibles para trabajar con un objeto de tipo apigis.Geocoder de Cortociudad (type: 'Cartociudad'):

candidates(parameters)

Geolocaliza tanto direcciones urbanas como localizaciones definidas por puntos kilométricos (PKs) asociados a carreteras. Sus argumentos son:

  • parameters <object > Parámetros del servicio. Los parámetros disponibles son los proporcionados por el propio servicio de Cartociudad para el servicio REST 'candidates'. Se pueden consultar en http://www.cartociudad.es/recursos/Documentacion_tecnica/CARTOCIUDAD_ServiciosWeb.pdf

find(parameters)

Geolocaliza una entidad y obteniene sus coordenadas geográficas. Sus argumentos son:

  • parameters <object > Parámetros del servicio. Los parámetros disponibles son los proporcionados por el propio servicio de Cartociudad para el servicio REST 'find'. Se pueden consultar en http://www.cartociudad.es/recursos/Documentacion_tecnica/CARTOCIUDAD_ServiciosWeb.pdf

reverse(parameters)

Calcula una dirección a partir de un par de coordenadas geográficas introducido en el sistema de referencia ETRS89. Sus argumentos son:

  • parameters <object > Parámetros del servicio. Los parámetros disponibles son los proporcionados por el propio servicio de Cartociudad para el servicio REST 'reverse'. Se pueden consultar en http://www.cartociudad.es/recursos/Documentacion_tecnica/CARTOCIUDAD_ServiciosWeb.pdf
