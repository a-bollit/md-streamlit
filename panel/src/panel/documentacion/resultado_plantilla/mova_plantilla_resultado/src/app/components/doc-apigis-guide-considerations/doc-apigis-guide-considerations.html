Consideraciones Generales sobre el API GIS

El API GIS proporciona clases para facilitar la estandarización y la generación de visores de mapas en las aplicaciones.

Para poder visualizar capas de información geográfica, estas capas siempre deberán estar contenidas dentro de un mapa. Este mapa, a su vez se deberá instanciar dentro de una vista. Además, el API GIS incluye herramientas que permiten la intercción con esas capas y otras funcionalidades.

Los puntos de entrada al API GIS son los siguientes:

  • ApiLayer

    • Punto de entrada para la generación de servicios. La documentación sobre este punto está disponible en doc-apigis-layers.

  • ApiMap

    • Punto de entrada para la generación de mapas. Un mapa puede estar formado por uno o más servicios. La documentación sobre este punto está disponible en doc-apigis-map.

  • ApiViewer

    Punto de entrada para la generación de visores. Un visor está compuesto de uno o más mapas. La documentación sobre este punto está disponible en doc-apigis-viewer.

  • ApiSimpleMarkerSymbol

    • Punto de entrada para la generación de simbología puntual. La documentación sobre este punto está disponible en doc-apigis-simplemarkersymbol.

  • ApiSimpleLineSymbol

    • Punto de entrada para la generación de simbología lineal. La documentación sobre este punto está disponible en doc-apigis-simplelinesymbol.

  • ApiSimpleFillSymbol

    • Punto de entrada para la generación de simbología poligonal. La documentación sobre este punto está disponible en doc-apigis-simplefillsymbol.

  • ApiFilter

    • Punto de entrada para la generación de filtros. La documentación sobre este punto está disponible en doc-apigis-filters.

  • ApiLoader

    • Punto de entrada para la carga de un visor completo mediante un fichero de configuración. La documentación sobre este punto está disponible en doc-apigis-loader.

  • ApiUtils

    • Punto de entrada para el uso de herramientas genéricas. La documentación sobre este punto está disponible en doc-apigis-utils.

  • ApiGeocoder

    • Punto de entrada para el uso de herramientas de geocodificación. La documentación sobre este punto está disponible en doc-apigis-geocoders.

La forma de integrar estos puntos de entrada en las aplicaciones se puede consultar en doc-apigis-integration.

Además, para poder emplear el API GIS, es necesario que se disponga de dependencias muy pesadas, que no han sido incluidas dentro de los paquetes del mismo. Es responsabilidad del integrador incluirlas dentro de sus aplicaciones. La forma de integrar estas dependencias se puede consultar en doc-apigis-dependencies.
