Consideraciones generales sobre Capas Operacionales

Las Capas Operaciones son un tipo especial de capas que se emplean para representar dibujados, resultados de búsquedas, identificaciones...

Este tipo de capas no se suelen generar con el contructor como el resto de capas, sino que se suelen añadir directamente al mapa en el que deseamos emplearlas con la función addOverlayLayer() tal y como se explica en la documentación general sobre apigis.Map.

IMPORTANTE

Para poder visualizar una capa, deberá estar contenida dentro de un objeto de tipo apigis.Map.

Así mismo, un objeto de tipo apigis.Map deberá estar contenido dentro de un objeto de tipo apigis.Viewer.


Se puede consultar información más detallada y ejemplos sobre el uso de apigis.Layer en el documento de Arquitectura sobre el API GIS.

Métodos Métodos comunes a todas las capas cuyo mapType: 'OL', typeLayer: 'Vector' y subtypeLayer: 'Overlay'.

Se muestran a continuación los métodos comunes a todas las capas operaciones de tipo apigis.Layer cuyo mapType: 'OL', typeLayer: 'Vector' y subtypeLayer: 'Overlay':


setSymbology(sld)

Permite establecer una simbología a la capa mediante el paso de un SLD. Sus argumentos son:

  • value <apigis.SimpleMarkerSymbol | apigis.SimpleFillSymbol | apigis.SimpleLineSymbol | apigis.SimpleMarkerSymbol[] | apigis.SimpleFillSymbol[] | apigis.SimpleLineSymbol[] | Function > Valor de la simbología a establecer.

identify(featureGeoJSON, identifyParams = {})

Permite ejecutar una consulta de identificación en cliente (navegador). Sus argumentos son:

  • featureGeoJSON <string > Feature en geojson sobre la que ejecutar la identificación.
  • identifyParams <object > Parámetros de identificación. Estos pueden ser:
    • filter <apigis.Filter > Filtro aplicado sobre la capa.
    • tolerance <number > Tolerancia en píxeles.

find(filterObj)

Ejecuta una búsqueda sobre el servicio en cliente (navegador). Sus argumentos son:

  • filterObj <apigis.Filter >Filtro. El objeto tendrá propiedades que harán referencia a las diferentes expresiones de la consulta.

addFeatures(featuresGeoJSON, symbology, color)

Permite añadir features a la capa. Sus argumentos son:

  • featuresGeoJSON <string | string[] >Colección de features a añadir. Acepta una feature, una colección de features o un array de features, todo ello en formato GeoJSON.
  • symbology <apigis.SimpleMarkerSymbol | apigis.SimpleFillSymbol | apigis.SimpleLineSymbol | apigis.SimpleMarkerSymbol[] | apigis.SimpleFillSymbol[] | apigis.SimpleLineSymbol[] | Function > Valor de la simbología a establecer a las features. En su defecto aplica la simbología de la capa.
  • color <string | number[] > Color a aplicar a las features.

rotateFeatures(featuresIds, angle = 0, group = false)

Permite rotar features de manera individual o grupal. Sus argumentos son:

  • featuresIds <string[] >Identificadores de las features que desean rotar.
  • angle = 0 <number > Grados de rotación.
  • group = false <boolean >ndica si la rotación se efecturá de manera grupal o individual.

addBuffer(geojsonObject, distance = 0.1, callback)

Permite añadir una geometría de buffer a la capa, a partir de una feature. Sus argumentos son:

  • geojsonObject <string > Feature en formato GeoJSON.
  • distance = 0.1 <number > Extensión del buffer./li>
  • callback <Function > Función de callback.

getFeatures()

Recupera las features de la capa.


getFeatureById(id)

Recupera la feature con el ID especificado en la capa.


clear()

Limpia la capa de features.


removeFeature(id)

Elimina la feature con el ID especificado en la capa. Sus argumentos son:

  • string <id > Identificador de la feature.

moveLayer(indexTo)

Cambia la posición de la capa dentro del mapa. Sus argumentos son:

  • indexTo <number > Nuevo índice en el mapa.

setCoordinatesToFeature(idFeature, coordinates)

Configura una nueva coordenada a una feature de la capa. Sus argumentos son:

  • idFeature <string > Identificador de la feature.
  • coordinates <number[] > Nueva coordenada.

translateOnClick(callback)

Permite activar el evento de traslación en la capa para mover features. La acción se desencadena con un clic sobre la feature. Sus argumentos son:

  • callback <Function > Función de callback.

translateAllFeatures(callback)

Permite activar el evento de traslación de todas las features de la capa. La acción se desencadena con un clic sobre la feature. Sus argumentos son:

  • callback <Function > Función de callback.

translateFeatures(callback)

Permite activar el evento de traslación de las features especificadas como parámetro. Sus argumentos son:

  • featuresIds <string[] > Identificadores de las features a trasladar.
  • callback <Function > Función de callback.

refresh()

Refresca la capa.
