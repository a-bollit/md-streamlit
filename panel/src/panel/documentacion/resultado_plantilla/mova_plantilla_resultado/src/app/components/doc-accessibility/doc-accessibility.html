Información Información sobre accesibilidad.

La accesibilidad en la web tiene como propósito garantizar el acceso y uso por personas con todo tipo de conocimientos y capacidades. MOVA proporciona configuración para cumplir este objetivo.

Documentación sobre accesibilidad en Angular: . https://codelabs.developers.google.com/angular-a11y?hl=es-419#0

Propiedad title Configuración del title en MOVA.

MOVA permite la configuración del title de varias formas.

Si queremos tener el mismo title para todas las vistas (configuración no recomendada de cara a la accesibilidad) se hará con la etiqueta title del archivo index.html, como en cualquier web:

Si queremos tener un title único para cada vista (configuración recomendada de cara a la accesibilidad) se hace con la propiedad appTitle del archivo config-default.json:

Lo que se define en este archivo es un prefijo personalizado para nuestra app, que se mostrará en el title de todas las vistas. Después de este prefijo se concatenará el nombre del estado correspondiente a la vista en la que nos encontremos en cada momento.

Foco

Se recomienda que en todos los componentes accionables, se distinga claramente cuando tiene el foco en el mismo o no.

Siguiendo este principio, todos los componentes accionables de MOVA cambian de tonalidad cuando tienen el foco.

Propiedades formulario Propiedades de formulario.

El aspecto más importante de cara a la accesibilidad son los formularios, ya que requiere una alta interacción con el usuario. MOVA proporciona funcionalidad en sus componentes para mejorar la accesibilidad de los formularios.

Para ver ejemplos de formularios, ir a ejemplo de formularios

Aria-label

Esta propiedad permite que los lectores distingan los distintos componentes de formulario (input, textarea, select...) con el campo concreto que representen (ciudad, nombre, edad...)

Aria-invalid

Esta propiedad permite que los lectores distingan cuándo los componentes de formulario contienen algún error, para que el usuario tenga conocimiento de ello.
