Autocompletar.
code
La propiedad autocomplete permite que el navegador autocomplete el contenido de un input con valores ya introducidos anteriormente. Por ejemplo, si el navegador tiene almacenado nuestro correo, se podrá autocompletar cualquier input que tenga la propiedad autocomplete con valor "email".

Para ver los valores que acepta la propiedad y tener más información ver la documentación oficial: https://developer.mozilla.org/en-US/docs/Web/HTML/Attributes/autocomplete

refAutocomplete
Autocompletar con valores.
code
La propiedad refAutocomplete permite definir una serie de valores para autocompletar el contenido del input, permitiendo filtrar si así se desea.

Para definir un input con autocompletado hay que definir un componente mv-input y un mat-autocomplete de Material https://material.angular.io/components/automcomplete/overview

HTML
TS

  <mv-input
    label="Label del input"
    appearance="legacy"
    matAutocomplete="auto">
  </mv-input>

  <mat-autocomplete #auto="matAutocomplete" [displayWith]="displayFn">
    <mat-option *ngFor="let option of filteredOptions | async" [value]="option">
      {{option.name}}
    </mat-option>
  </mat-autocomplete>

  <mv-input
    [refAutocomplete]="auto2"
    [formControl]="optionsCtrl">
  </mv-input>

  <mat-autocomplete #auto2="matAutocomplete">
    <mat-option
      *ngFor="let option of filteredOptions | async"
      [value]="option.name"
    >
      {{option.name}}
    </mat-option>
  </mat-autocomplete>

  <mv-input
    [refAutocomplete]="auto5"
    [formControl]="optionsGroupCtrl">
  </mv-input>

  <mat-autocomplete #auto5="matAutocomplete">
    <mat-optgroup
      *ngFor="let group of filteredGroupOptions | async"
      [label]="group.index"
    >
      <mat-option *ngFor="let name of group.names" [value]="name">
        {{name}}
      </mat-option>
    </mat-optgroup>
  </mat-autocomplete>

keyupEvent
Evento keyup.
code
El evento keyupEvent permite definir una función que se ejecutará cuando el usuario deja de teclear

La diferencia con el evento changeEvent es que no se ejecutará la función keyupEvent hasta que el usuario deje de teclear, de manera que si mantiene pulsado de manera continuada un valor del teclado se estará introduciendo muchos caracteres sin que se ejecute la función keyupEvent, sin embargo sí se estaría ejecutando la función changeEvent porque el valor sí está cambiando.
